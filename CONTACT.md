## 🐛 SIGNALER UN BUG

### Informations à fournir

Pour un traitement rapide, incluez :

1. **Description du bug**
   - Que s'est-il passé ?
   - Qu'attendiez-vous ?

2. **Étapes pour reproduire**
   - Étape 1 : ...
   - Étape 2 : ...
   - Étape 3 : ...

3. **Environnement**
   - Navigateur : Chrome 120 / Firefox 121 / Safari 17 / Edge 120
   - OS : Windows 11 / macOS 14 / Ubuntu 22.04
   - Version de l'application : 1.0.0

4. **Captures d'écran**
   - Joignez des captures d'écran si possible

5. **Message d'erreur**
   - Copiez le message d'erreur exact
   - Ouvrez la console (F12) et copiez les erreurs

6. **Logs**
   - Console navigateur (F12 → Console)

## 🤝 CONTRIBUER

### Comment contribuer

1. **Fork le projet**
   ```bash
   git clone https://github.com/votre-org/made-ong-humanitarian.git
   ```

2. **Créer une branche**
   ```bash
   git checkout -b feature/ma-fonctionnalite
   ```

3. **Faire vos modifications**
   - Suivez les conventions de code
   - Ajoutez des tests si possible
   - Documentez vos changements

4. **Commit**
   ```bash
   git commit -m "feat: Ajout de ma fonctionnalité"
   ```

5. **Push**
   ```bash
   git push origin feature/ma-fonctionnalite
   ```

6. **Créer une Pull Request**
   - Décrivez vos changements
   - Expliquez pourquoi c'est utile
   - Ajoutez des captures d'écran si pertinent

### Conventions de commit

```
feat: Nouvelle fonctionnalité
fix: Correction de bug
docs: Documentation
style: Formatage (pas de changement de code)
refactor: Refactoring
test: Ajout de tests
chore: Maintenance
```

### Code de conduite

- Soyez respectueux
- Soyez constructif
- Aidez les autres
- Partagez vos connaissances

## 🚨 URGENCES

### Définition d'une urgence

Une urgence est un problème qui :
- Empêche complètement l'utilisation de l'application
- Cause une perte de données
- Compromet la sécurité
- Affecte tous les utilisateurs

### Procédure d'urgence

1. **Envoyez un email immédiatement**
   - **Email** : Made711@gmail.com
   - **Objet** : [URGENT] Description du problème
   - **Contenu** : Description détaillée + impact

2. **Décrivez l'impact**
   - Nombre d'utilisateurs affectés
   - Fonctionnalités bloquées
   - Perte de données potentielle

3. **Actions temporaires**
   - Notez l'heure du problème
   - Prenez des captures d'écran
   - Notez les messages d'erreur
   - Ne tentez pas de "réparer" vous-même

4. **Attendez la réponse**
   - Délai : 2-4 heures (jours ouvrables)
   - Délai : 24 heures (week-end)
